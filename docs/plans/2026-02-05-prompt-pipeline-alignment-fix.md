# Prompt Pipeline Alignment Fix -- Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Fix the broken prompt management pipeline so that every GENERATED/HYBRID field in every template type has a seeded prompt in the database and shows as "defined" in the dashboard, with the ability to create missing prompts from the UI.

**Architecture:** The pipeline has 5 components that must agree on field names: field registries (.md) -> field_row_mappings.json -> template_fields.py -> prompt_manager.py -> seed_prompts.py -> DB -> API -> Frontend. The root cause is key name mismatches between field_row_mappings.json and prompt_manager.py, missing prompt definitions for 50-70% of fields, a hardcoded disabled Create button, and EXTRACTED/STATIC fields inflating the "missing" count. We fix the canonical names in prompt_manager.py to match field_row_mappings.json (the source of truth), add all missing prompt definitions, wire the Create button, and filter EXTRACTED/STATIC fields from coverage stats.

**Tech Stack:** Python 3.11, FastAPI, SQLAlchemy 2.0, React 19, TypeScript, TailwindCSS

**Root Causes Addressed:**
1. Key name mismatches between field_row_mappings.json and prompt_manager.py
2. Missing prompt definitions for GENERATED/HYBRID fields
3. Greyed-out Create button in SectionGroup.tsx (hardcoded `disabled`)
4. EXTRACTED/STATIC fields inflating "missing" count in dashboard

Write a throwaway script to identify mismatches:

```python
# Save as backend/scripts/_audit_prompt_keys.py
import json
import sys
from pathlib import Path

sys.path.insert(0, str(Path(__file__).parent.parent))
from app.services.prompt_manager import PromptManager

pm = PromptManager()
defaults = pm.get_default_prompts()

with open(Path(__file__).parent / "field_row_mappings.json") as f:
    mappings = json.load(f)

for template_type, fields in mappings.items():
    generated_hybrid = [
        name for name, data in fields.items()
        if data.get("field_type") in ("GENERATED", "HYBRID")
    ]

    print(f"\n=== {template_type.upper()} ===")
    print(f"GENERATED/HYBRID fields: {len(generated_hybrid)}")

    missing = []
    for field_name in generated_hybrid:
        template_key = f"{template_type}:{field_name}"
        if template_key not in defaults and field_name not in defaults:
            missing.append(field_name)

    print(f"Missing prompts: {len(missing)}")
    for m in sorted(missing):
        print(f"  - {m}")

    # Also find prompt keys that don't match any field
    prefix = f"{template_type}:"
    prompt_keys = [k.replace(prefix, "") for k in defaults if k.startswith(prefix)]
    orphaned = [k for k in prompt_keys if k not in fields]
    if orphaned:
        print(f"Orphaned prompt keys (no matching field): {len(orphaned)}")
        for o in sorted(orphaned):
            print(f"  - {o}")
```

```python
"""Validate prompt coverage: every GENERATED/HYBRID field must have a prompt defined."""
import json
from pathlib import Path

import pytest

from app.services.prompt_manager import PromptManager
from app.services.template_fields import TEMPLATE_FIELD_REGISTRY, FieldType


@pytest.mark.unit
class TestPromptCoverage:
    """Ensure prompt_manager defines prompts for all generatable fields."""

    @pytest.fixture(autouse=True)
    def setup(self):
        self.pm = PromptManager()
        self.defaults = self.pm.get_default_prompts()

    @pytest.mark.parametrize("template_type", [
        "aggregators", "opr", "mpp", "adop", "adre", "commercial"
    ])
    def test_all_generated_hybrid_fields_have_prompts(self, template_type):
        """Every GENERATED/HYBRID field must have a matching prompt."""
        fields = TEMPLATE_FIELD_REGISTRY[template_type]
        missing = []

        for field_name, field_def in fields.items():
            if field_def.field_type not in (FieldType.GENERATED, FieldType.HYBRID):
                continue

            template_key = f"{template_type}:{field_name}"
            if template_key not in self.defaults and field_name not in self.defaults:
                missing.append(field_name)

        assert missing == [], (
            f"{template_type}: {len(missing)} GENERATED/HYBRID fields missing prompts: "
            f"{', '.join(sorted(missing))}"
        )

    @pytest.mark.parametrize("template_type", [
        "aggregators", "opr", "mpp", "adop", "adre", "commercial"
    ])
    def test_no_orphaned_prompt_keys(self, template_type):
        """Every prompt key should correspond to a field in the registry."""
        fields = TEMPLATE_FIELD_REGISTRY[template_type]
        prefix = f"{template_type}:"
        prompt_keys = [
            k.replace(prefix, "") for k in self.defaults if k.startswith(prefix)
        ]
        orphaned = [k for k in prompt_keys if k not in fields]

        assert orphaned == [], (
            f"{template_type}: {len(orphaned)} prompt keys with no matching field: "
            f"{', '.join(sorted(orphaned))}"
        )
```

## Risk Notes

1. **Database re-seeding with --force overwrites manual edits.** If anyone has manually edited prompts in the DB via the dashboard, those edits will be lost. Consider backing up the prompts table first: `pg_dump -t prompts > prompts_backup.sql`

2. **prompt_manager.py will grow significantly.** With 6 templates x 60-100 GENERATED/HYBRID fields each, the file could reach 2000+ lines. A future refactor could move each template's prompts into separate files (e.g., `prompts/aggregators.py`), but that is out of scope for this fix.

3. **The prompt documents in prompt-organizaton/ are the source of truth for prompt CONTENT.** prompt_manager.py should mirror them. Any future prompt edits should update both the .md files and prompt_manager.py defaults, or (better) the database-backed prompt management should replace hardcoded defaults entirely.
