## Introduction

### Who Is This Guide For?

This guide is for **Marketing Managers** who are responsible for:
- Reviewing AI-generated property content
- Ensuring content quality and brand consistency
- Approving or requesting revisions
- Maintaining content standards across all websites

As the quality gatekeeper, you ensure every piece of content published meets our high standards and accurately represents each property.

### What You'll Learn

By the end of this guide, you'll be able to:
- Efficiently review projects in the approval queue
- Identify content issues and request specific revisions
- Approve high-quality projects quickly
- Use bulk approval for time efficiency
- Track approval metrics and backlog

## Getting Started

### Your Role in the Workflow

```
Content Creator → Marketing Manager → Publisher → Live Website
                       (YOU)
```

You are the quality checkpoint between content creation and publishing.

### Key Responsibilities

1. **Review Quality** - Check content accuracy and professionalism
2. **Verify Data** - Ensure pricing, dates, and details match source PDF
3. **Brand Consistency** - Maintain tone and style standards
4. **Timely Review** - Target 24-hour turnaround for approvals

- **Email:** @your-domain.com Google Workspace account
- **Role:** Marketing Manager role assigned by admin
- **Browser:** Chrome, Firefox, Safari, or Edge (latest version)

## Overview of Approval Process

### The Approval Workflow

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  1. Content Creator submits project                     │
│           ↓                                             │
│  2. Project enters your Approval Queue                  │
│           ↓                                             │
│  3. You review content and data                         │
│           ↓                                             │
│  4. You make decision:                                  │
│           ├─→ APPROVE → Publishing Queue                │
│           ├─→ REQUEST REVISION → Back to Creator        │
│           └─→ REJECT → Archived                         │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### Decision Criteria

**APPROVE when:**
- All content is accurate and professional
- Data matches source PDF
- SEO requirements met (character counts, etc.)
- Images are appropriate and high quality
- No brand inconsistencies

**REQUEST REVISION when:**
- Minor errors that content creator can fix
- Missing information
- Content needs improvement
- Data discrepancies

**REJECT when:**
- Project is duplicate
- Wrong property uploaded
- Unsalvageable quality issues
- Project cancelled

## Requesting Revisions

### When to Request Revisions

Request revisions when:
- Minor data inaccuracies
- Content needs improvement
- Missing information
- Character count issues
- Image quality concerns
- Spelling/grammar errors

**Note:** Only request revisions for fixable issues. If the project is fundamentally flawed, reject it instead.

### Revision Steps

**Step 1:** Click **"Request Revision"** button

**Step 2:** Provide specific, actionable feedback:

```
┌──────────────────────────────────────────────────────────┐
│ Revision Request Comments                                │
├──────────────────────────────────────────────────────────┤
│ [Please make the following changes:                      │
│                                                          │
│  1. Update starting price to AED 1.35M (not 1.2M).      │
│     Check page 7 of PDF for correct pricing.            │
│                                                          │
│  2. Handover date should be Q1 2027, not Q4 2026.       │
│     Developer announced delay last week.                 │
│                                                          │
│  3. Meta description is too generic. Add specific       │
│     mention of waterfront location and Marina views.    │
│                                                          │
│  4. Overview section - remove duplicate paragraph       │
│     about payment plan (appears twice).                 │
│                                                          │
│ Please resubmit after making these updates. Thanks!]    │
│                                                          │
│ [Cancel]  [Submit Revision Request]                      │
└──────────────────────────────────────────────────────────┘
```

**Best practices for revision requests:**

✅ **Good feedback:**
- Specific: "Update starting price to AED 1.35M"
- Actionable: "Check page 7 of PDF"
- Numbered: Easy to follow
- Explains why: "Developer announced delay"

❌ **Poor feedback:**
- Vague: "Fix the pricing"
- Unclear: "Make it better"
- No guidance: "This is wrong"

**Step 3:** Submit revision request

The project returns to the content creator with your comments.

## Rejecting Projects

### When to Reject

Reject a project when:
- Project is a duplicate
- Wrong property uploaded entirely
- Project has been cancelled by developer
- Quality issues are unsalvageable
- Not appropriate for the website

**Note:** Rejection is permanent. Use sparingly.

### Rejection Steps

**Step 1:** Click **"Reject Project"** button

**Step 2:** Select rejection reason:

```
┌──────────────────────────────────────────────────────────┐
```

## Bulk Approval

For efficiency when you have multiple high-quality projects.

### When to Use Bulk Approval

Use bulk approval when:
- Multiple projects from same creator
- All projects meet quality standards
- Similar property types (e.g., all OPR residential)

**Warning:** Only use bulk approval for projects you've individually reviewed. Don't approve projects sight-unseen.

## Best Practices

### 1. Use Templates for Common Feedback

Save time with templated revision requests:

**Template: Pricing Update**
```
"Please update starting price to AED [AMOUNT].
Check page [PAGE] of PDF for correct pricing."
```

**Template: Meta Description Too Long**
```
"Meta description exceeds 160 characters ([COUNT]/160).
Please shorten while keeping key selling points."
```

**Template: Missing Information**
```
"Missing [FIELD]. Please check PDF page [PAGE] and add this information."
```

### 2. Review in Batches

Group similar projects:
- All OPR projects together
- All projects from same creator
- All projects from same developer

This helps maintain consistency.

### 3. Maintain Brand Voice

Ensure all content matches our brand standards:

**Aggregators (24+ portals):**
- Clear and factual
- Portal-friendly formatting
- SEO-optimized keywords

**OPR (opr.ae):**
- Professional yet approachable
- Benefit-focused
- Emphasize lifestyle and investment

**MPP (main-portal.com):**
- Premium and sophisticated
- Luxury market positioning
- Aspirational tone

**ADOP (abudhabioffplan.ae):**
- Abu Dhabi market focus
- Investment opportunity emphasis
- Professional tone

**ADRE (secondary-market-portal.com):**
- Abu Dhabi community focus
- Lifestyle and location emphasis
- Trustworthy tone

**Commercial (cre.main-portal.com):**
- Business-focused
- ROI and yield emphasis
- Professional investor tone

### 4. Be Constructive

When requesting revisions:
- Be specific and helpful
- Explain why changes are needed
- Acknowledge what's good
- Provide examples when possible

✅ Good: "Great work on the overview! Please update the starting price to AED 1.35M (see PDF page 7)."

❌ Bad: "Wrong price. Fix it."

### 5. Track Common Issues

Keep notes on patterns:
- Which creators need more training?
- Which developers provide poor PDFs?
- Which fields most often need revision?

Share insights with team to improve overall quality.

### 6. Use Two-Monitor Setup

Efficiency tip:
- Monitor 1: PDP Automation system
- Monitor 2: Original PDF open

Makes cross-referencing much faster.

## Common Issues & Solutions

### Issue 1: Can't Access PDF

**Problem:** "View Original PDF" link doesn't work

**Solution:**
1. Ask content creator to re-upload PDF
2. Or request PDF via email/Slack
3. Can still review content, just can't cross-reference

---

### Issue 2: Creator Hasn't Responded to Revision

**Problem:** Revision requested 3 days ago, no response

**Solution:**
1. Send reminder via Slack or email
2. If urgent, re-assign to another creator
3. If very urgent, make edits yourself and approve

---

### Issue 3: Disagreement on Content Quality

**Problem:** Creator thinks content is fine, you disagree

**Solution:**
1. Provide specific examples of issues
2. Reference brand guidelines
3. Escalate to senior marketing manager if needed
4. Document the decision

---

### Issue 4: Too Many Projects in Queue

**Problem:** Queue backlog growing, can't keep up

**Solution:**
1. Use bulk approval for similar high-quality projects
2. Focus on overdue items first
3. Delegate to other marketing managers if available
4. Request pause on new uploads temporarily

---

### Issue 5: Quality Declining

**Problem:** Recent projects have more errors than usual

**Solution:**
1. Check if new content creator needs training
2. Review if prompt changes affected output
3. Provide feedback to admin team
4. Schedule refresher training

## FAQs

**Q: How long should I spend reviewing each project?**

A: Average 10-15 minutes per project. Simple projects may take 5 minutes, complex ones up to 20 minutes.

---

**Q: What if I'm unsure about data accuracy?**

A: Check the original PDF. If still unsure, request clarification from content creator or sales team.

---

**Q: Can I edit content myself instead of requesting revision?**

A: Yes, but only for minor edits (typos, small tweaks). For substantial changes, request revision so creator can learn.

---

**Q: What's the difference between revision and rejection?**

A: Revision = fixable issues, project returns to creator. Rejection = unfixable or cancelled, project archived.

---

**Q: How do I know if images are high quality enough?**

A: Images should be clear, not pixelated when viewed at full size. If you can see visible compression or blur, request better quality.

---

**Q: What if the AI-generated content is completely wrong?**

A: Request revision with specific corrections. If AI consistently fails on certain projects, report to admin to adjust prompts.

---

**Q: Can I approve a project with QA warnings?**

A: Yes, but review the warnings carefully. Some are critical (character count violations), others are suggestions.

---

**Q: What's the SLA for approvals?**

A: Target is 24 hours. Overdue projects are flagged in the queue.

---

**Q: How do I handle urgent/priority projects?**

A: Admin can flag projects as high-priority. These appear at top of queue. Review these first.

---

**Q: What if I accidentally approved the wrong project?**

A: Contact admin immediately. If not yet published, project can be recalled from publishing queue.
